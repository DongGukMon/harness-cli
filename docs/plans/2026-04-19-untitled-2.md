**Goal:** Activate the existing Phase 2 slot in the light harness flow so Codex reviews the combined design doc before implementation (P5) runs, preventing costly post-impl REJECT → re-impl chains.

**Architecture:** Touch `src/config.ts` (add P2 preset + gate-aware retry limit), `src/state.ts` (flip P2 from `'skipped'` to `'pending'` for new light runs only), `src/context/assembler.ts` (new `FIVE_AXIS_DESIGN_GATE_LIGHT` rubric + light branch in `buildGatePromptPhase2`), and `src/phases/runner.ts` (pass `phase` to `getGateRetryLimit` at two call sites). All existing full-flow paths are byte-identical before and after.

**Tech Stack:** TypeScript, Vitest, existing harness infrastructure (`src/config.ts`, `src/state.ts`, `src/context/assembler.ts`, `src/phases/runner.ts`).
